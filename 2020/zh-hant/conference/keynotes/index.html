基調演講

機器學習時代下的 Python

  • 自我介紹
  • 演講
  • Slido

自我介紹

Peter Wang 擁有康奈爾大學物理學學士學位，從事商業科學計算和可視化軟件的開發已超過 15 年。在程式設計及開發方面擁有非常豐富的經驗，涉及的領域亦非常廣泛，其中包含 3D 圖形、地球物理、大數據模擬和可視化、財務風險建模和醫學成像等。

因在向量計算和互動式可視化基本原理的興趣，使他成為 Anaconda 共同創辦人（過去稱為 Continuum Analytics），並領導開源及社群創新小組。

作為 PyData 社群和會議的發起者，他投入大量時間和精力發展 Python 資料科學社群，在世界各地的研討會上倡導和傳授 Python。

演講

Peter 將會談談他認為 Python 變得熱門的原因。包含 Python 在數據分析領域的優缺點，以及為什麼是個自由開放原始碼軟體、無障礙(初學者友善)語言，及 Python 在機器學習和人工智慧趨勢潮流的世界裡，如何有著不可取代的地位。

新世代的教學思維 - 從機器學習線上平台與入門課程談起

  • 自我介紹
  • 演講
  • Slido

自我介紹

蘇文鈺現為成功大學資訊工程系教授，國立交通大學士，紐約大學坦登學院碩博士，史丹佛大學博士後研究，工研院電通所工程師，中華大學資訊工程系副教授。2011 年之前致力於神經網路訊號處理，生理訊號分析，視音訊處理器設計，多核心處理器設計，內嵌記憶體管理與電子系統層工具 OpenESL，2012 年之後轉為致力於音樂音訊處理，機器學習與虛擬管弦樂團。多年來，以提高學生實務實作能力為要旨。
2013 年，有鑒於資訊科技與程式設計日益重要，學生的實作探究能力降低，成立 Program The World Association (PTWA)（社團法人中華民國愛自造者學習協會），2014 年從嘉義東石偏鄉開始，帶領團隊教導偏鄉弱勢中小學學生以程式設計。截至 2020 年，摩頂放踵，足跡遍及 12 個縣市，輔導的學校與非營利組織超過 70 個，受惠的國教學生與教師超過 2500 人。多年來，課程漸趨多元，教學計劃包含 Train The Trainer，看見家鄉，程式與自造巡迴課程，自走車競賽，Coding-X 偏鄉高中職生程式設計，樂塾，特教科技課程。每年舉辦活動超過 70 場。並積極參與成功大學不分系，弱勢招生，上機考特殊選才，線上課程與考試系統等等促進高教進步的工作．多年來，PTWA 以募捐所得進行免費公益教學，雖感艱辛，但是見到弱勢家庭孩子得以成長，頗為欣慰。
2018 年，有鑒於人工智慧的日益重要，但是多數課程缺乏實務操作經驗，吸引學生投入之不足，仿照國外名校，投入開發以遊戲打怪過關的方式的課程與線上平台 - ML Game，2020 年，平台完成，並在成功大學資訊系順利開課，目前已有多所學校接洽推廣，並預計在 2020/2021 與台灣微軟合作推廣課程，並舉辦全國性機器學習比賽。
除了一般學術著作外，另出版「做孩子的重要他人」與「老師是孩子遲來的父母」，紀錄多年在偏鄉與弱勢家庭的孩子的輔導工作。

演講

隨著新冠病毒，全世界的工作型態與教學方式劇烈地改變，許多知名的世界級公司如 Google 與 Facebook 都陸續宣布 WFH(Work From Home) 會至少持續到明年為止。
國立成功大學在 2020/2 也宣布超過 100 人的課程要改用線上教學，許多教授緊急準備線上教學，但是多數教授除了錄製教學影片之外往往只能搭配學校的 moodle 來輔助學生的學習。
我開設的「遊戲式機器學習入門」為大學部選修課，也在上述要求下開始準備線上教學。因為風聞線上教學的效果不佳，所以我的研究生與我，加上一位預先申請的業界專家，共同思考如何度過這學期，同時希望教學成效不要打太多折。
除了預錄教學影片外，我們使用了數個線上應用軟體與社群，並且共筆 FAQ 盡可能回答各式問題外，還架設了一個線上競賽平台，讓學生在考試日可以上傳程式後以競賽得分作為主要評量方式，並直播比賽過程。
除說明這門課的上課方式之外，也會介紹此平台的使用方式與未來的發展，同時也深深感到，身為老師要為了世界與新世代學生的改變而變更自己的教學思維與方式，以維持台灣的競爭力。

哎呀! 我成為了開源專案的維護者! 😱

自我介紹

Mariatta 是 Python 核心開發人員、Zapier 的軟體工程師、溫哥華 PyLadies 共同創辦人，也是 PyCascades 會議的創始成員之一。她在約二十年前移居加拿大，現在與丈夫和兩個孩子一起住在溫哥華。在業餘時間，她致力於開源專案、打造 GitHub 機器人、修正錯字，並喜歡向你介紹 f-strings。而她最喜歡的表情符號是😝。

演講

在開源程式碼的世界裡，我自認還相對陌生。我貢獻的第一個開源專案是在2016年的夏天，在參與不久後，我發現我獲得了可以在別人的開源專案裡提交 (commit) 的權限。成為開源專案的維護者帶來一系列我還沒完全準備好的獨特挑戰。在本次演講中，我將分享我的旅程和我在過程中學到的事情，並為其他有抱負的開源維護者和貢獻者提供一些建議。
