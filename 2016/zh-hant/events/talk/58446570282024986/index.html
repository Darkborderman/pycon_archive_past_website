Boost Maintainability

摘要

“To understand a random line, how many lines do you need to read back?” is the core concept of this talk. This talk will introduce you the practicing techniques from years’ experience to maximize the maintainability.

「為了瞭解程式中隨機的一行，你需要往回讀多少行？」這是這場演講的核心概念。這場演講會介紹多年開發經驗累積而來的實踐技巧，來最大化可維護性。

說明

In your code base, to understand a random line, how many lines do you need to read back? Cloud you make it zero? This talk will start with the impact of maintainability, define the maintainability as “to understand a random line, the lines you need to read back”, show the practicing techniques from years’ experience to make it zero, or nearly zero, and finally, reach the goal: boost the maintainability. 為了瞭解你寫的程式中隨機的一行，需要往回讀多少行？ 可以使之為零嗎？這場演講會從可維護性的重要性出發，將可維護性定義為「為了解隨機的一行，需要往回讀的行數」，介紹多年開發經驗累積而來的實踐技巧，使之為零，或接近零。最後達到我們的目標：boost maintainability。
